# 验收测试说明

软件包：wps-read-aloud-zhangjingyao
版本：1.0.13
目标环境：银河麒麟 V10 ARM64 + WPS 2023 for Linux

## 安装测试

```bash
sudo dpkg -i wps-read-aloud-zhangjingyao_1.0.13_arm64.deb
```

预期结果：
- 安装过程无错误退出。
- 安装输出包含服务健康检查、语音合成自检和播放器探测结果。
- 覆盖升级时，后台 `wps-tts.service` 会被重启到新版本。
- 安装日志写入 `/var/log/wps-read-aloud-install.log`。

## WPS 加载项测试

1. 重启 WPS。
2. 打开 WPS 文字。
3. 确认顶部出现“文档朗读”选项卡。
4. 确认没有额外打开右侧任务窗格。
5. 确认选项卡包含“全文朗读、当前位置朗读、选区朗读、暂停朗读、继续朗读、停止朗读、朗读语速、状态检查、关于朗读”。
6. 确认不再出现“朗读音量”按钮或菜单。
7. 点击“关于朗读”，确认可以看到加载项名称、开发者 `zhangjingyao`、版本 `1.0.13`、发布时间 `20260516` 和说明文件链接。

## 朗读功能测试

1. 打开包含多句中文正文的文档。
2. 分别点击“全文朗读”“当前位置朗读”“选区朗读”。

预期结果：
- 可以正常合成并播放语音。
- 当前正在朗读的完整语句在 WPS 文档中呈选中状态。
- 朗读进入下一句时，WPS 文档选区同步切换到下一句。
- 如果下一句在下一页，WPS 应滚动或翻页，使当前选中语句保持可见。
- 播放过程中可以暂停、继续、停止。
- 不出现“WPS 内置浏览器阻止了自动播放”的提示。
- 不出现 `Unexpected non-whitespace character after JSON` 这类原始技术错误。

## 语速测试

1. 打开“朗读语速”菜单。
2. 切换到 `0.8x`、`1.2x` 等不同选项。
3. 在朗读过程中切换语速。

预期结果：
- 当前语速选项有勾选或按下状态提示。
- 正在播放的当前句不强行变速。
- 从下一句开始按新语速重新合成并播放。
- 未播放的预合成缓存会被清理并按新语速重新生成。

## 音量测试

预期结果：
- 加载项内不提供音量调节。
- 系统音量、声卡音量或桌面环境音量设置可以控制实际播放音量。

## 中英文混合朗读测试

示例文本：

```text
这是一个 WPS read aloud test，用于验证中文和 English 混合朗读。
```

预期结果：
- 中文内容清晰朗读。
- 英文内容不会跳过，会按预处理后的字母间隔方式清晰拼读。
- 当前朗读语句仍在 WPS 文档中同步选中。

## 异常操作测试

1. 连续快速点击朗读按钮。
2. 连续快速点击停止、暂停、继续。
3. 朗读过程中再次点击“全文朗读”。
4. 朗读过程中关闭并重新打开 WPS。

预期结果：
- 不出现多段音频同时播放。
- 上一轮朗读会被停止或安全接管。
- 服务端不会残留长期运行的合成或播放进程。
- 再次打开 WPS 后可以继续使用朗读功能。

## 低性能机器测试

预期结果：
- 单次最多处理 1000 句，单句最多 1000 字，避免低配机器被超长文本拖垮。
- 服务端预合成后续句子以减少等待，但不会无限制生成全部音频。
- 停止按钮在合成或播放等待期间仍应能结束当前朗读。
